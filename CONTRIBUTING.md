# Contributing to Ved

Ved is an experimental systems programming project focused on deterministic
orchestration and reliability engineering for distributed software.

We welcome thoughtful contributions from engineers interested in runtime
architecture, language design, and control-plane systems.

## Before Contributing

Please:

- Read existing documentation and issues
- Search for related proposals or discussions
- Open an issue before implementing large changes
- Be prepared to explain architectural trade-offs

Early alignment reduces rework and improves design quality.

## Contribution Areas

Contributions may include:

- runtime architecture experiments
- compiler and IR design work
- documentation improvements
- developer tooling
- testing and validation infrastructure

## Design Expectations

Ved prioritizes:

- determinism and reproducibility
- clarity of system behaviour
- minimal accidental complexity
- long-term maintainability

Proposals should align with these principles.

## Pull Request Guidelines

- Keep changes focused and well-scoped
- Provide clear technical rationale
- Include documentation updates when relevant
- Avoid speculative features without prior discussion

Maintainers may request revisions before merging.

## Communication

Technical discussions should remain:

- respectful
- precise
- evidence-driven

Healthy disagreement is encouraged when grounded in reasoning.

## Project Status

Ved is in early development.  
Architectural decisions may evolve as core concepts are validated.

Thank you for contributing.
